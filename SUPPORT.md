# Support

## Where to Ask Questions

- **Usage questions** — open a GitHub Discussion once the project enables
  Discussions. Until then, use the `question` issue template.
- **Bug reports** — open a GitHub Issue using the `bug_report` template.
- **Feature requests** — open a GitHub Issue using the `feature_request`
  template. Please check existing issues first.
- **Security concerns** — do **not** use public issues. Follow the flow in
  [SECURITY.md](SECURITY.md).

## Response Expectations

This is a community-maintained project. Maintainers triage new issues at least
weekly, but response times are best-effort. Pull requests that include a clear
reproduction case and a regression test are the fastest path to a fix.

## Commercial Support

The project does not currently offer commercial support or SLA-bound response
times. If you need guaranteed turnaround, the usual path is to vendor the
library and maintain a local fork. Sponsorship channels may be opened in the
future (see `.github/FUNDING.yml`); until then, development is best-effort.
